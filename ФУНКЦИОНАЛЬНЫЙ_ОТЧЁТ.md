# Функциональный отчёт: Система обработки банковских квитанций (чекетоп)

**Дата отчёта:** 4 марта 2026  
**Версия:** 1.0

---

## 1. Общее описание системы

Система **чекетоп** — это набор инструментов для работы с PDF-документами банковских квитанций (в первую очередь Альфа-Банка). Основные задачи системы:

1. **Пересборка PDF** — создание «чистых» PDF из исходных без копирования страниц как бинарных объектов (подход «чистой комнаты»).
2. **Замена текстовых реквизитов** — подмена номеров счетов, сумм, имён, дат и идентификаторов в квитанциях.
3. **Соответствие forensic-требованиям** — минимизация следов редактирования, сохранение структуры исходного документа.
4. **Идентификация чеков** — справочная информация по распознаванию квитанций Альфа-Банка.

## 3. Модули системы

### 3.1. `rebuild_pdf.py` — ядро системы

**Назначение:** Clean Room PDF Generation — пересборка PDF путём прорисовки каждого визуального элемента (изображения + текстовые span'ы) на новом пустом документе. Бинарное копирование страниц не используется.

**Ключевые возможности:**
- Замена текста через `--replace OLD=NEW` (множественные замены)
- Замена из JSON: `--replacements-json file.json`
- Donor PDF для копирования стиля текста
- Работа со шрифтами: Tahoma/Arial, алиасы, сохранение исходных шрифтов
- Forensic-опции: `--strict-forensic`, `--native-structure`, `--ios-match`

**Основные опции CLI:**

| Опция | Описание |
|-------|----------|
| `input_pdf`, `output_pdf` | Входной и выходной файлы |
| `--replace OLD=NEW` | Текстовая замена (повторяемая) |
| `--replacements-json PATH` | Файл JSON с заменами |
| `--donor-pdf PATH` | PDF для копирования стиля текста |
| `--donor-text "..."` | Текст в donor для определения стиля (по умолчанию "1000 RUR") |
| `--strict-forensic` | Код возврата ≠0 при forensic-предупреждениях |
| `--no-size-optimize` | Отключить сжатие и оптимизацию размера |
| `--numbers-grow-left` | Суммы выравнивать по правому краю (числа растут влево) |
| `--preserve-font-metrics` | Не форсировать Tahoma, использовать исходные шрифты |
| `--match-input-pdf-version` | Версия PDF как у входного (по умолчанию вкл.) |
| `--copy-input-metadata` | Копировать метаданные из input (по умолчанию вкл.) |
| `--no-preserve-source-fonts` | Использовать Tahoma вместо исходных шрифтов |
| `--font-alias NAME` | Имя шрифта в ресурсах PDF |
| `--font-alias-from-input` | Имя шрифта из первого шрифта input |
| `--patch-header` | Удалить «Written by MuPDF» из заголовка |
| `--native-structure` | Структура как input ru.pdf (Im1/Im2/Im3, G1) |
| `--ios-match` | Полное совпадение с iOS-эталоном (BaseFont, Producer) |

**Внутренние компоненты:**
- `build_clean_room_pdf()` — основная логика пересборки
- `draw_page_images_from_xrefs()` — копирование изображений
- `draw_page_vector_paths()` — копирование векторных объектов (линии, прямоугольники)
- `draw_text_block()` — отрисовка текста с заменами
- `replace_text()` — токен-замена с границами слов (без каскадных эффектов)
- `_apply_native_structure_patch()` — пост-обработка для варианта C (fzImg→Im, Producer, Metadata)
- `forensic_self_check()` — проверка на подозрительные маркеры

### 3.2. `patch_pdf_header.py`

**Назначение:** Вариант А/Б — байтовая замена блока комментариев после `%PDF-1.x`, удаление «Written by MuPDF».

**Использование:**
```
python3 patch_pdf_header.py input.pdf output.pdf
```

Или как пост-обработка: `patch_file(path)` для файла по месту.

**Логика:** Замена MuPDF-строки на нейтральный iOS-стиль комментария с сохранением длины блока (offset'ы xref не меняются).

### 3.3. `content_stream_replace.py`

**Назначение:** Точечная замена текста в PDF без полной пересборки. Минимальные изменения: только целевой текст, метаданные и структура сохраняются.

**Использование:**
```
python3 content_stream_replace.py input.pdf output.pdf --replace "200=100"
python3 content_stream_replace.py input.pdf output.pdf --replace "200=100" --preserve-id
```

**Особенности:**
- Использует redaction (белая заливка + поверх новый текст)
- Опция `--preserve-id` восстанавливает оригинальный Document ID в trailer

### 3.4. `pdf_info.py`

**Назначение:** Извлечение информации о PDF: метаданные, шрифты, слои (OCG).

**Использование:**
```
python3 pdf_info.py input.pdf
python3 pdf_info.py input.pdf --fonts-from-text
```

**Выводит:**
- Метаданные (title, author, producer и т.д.)
- Шрифты из ресурсов страниц
- Шрифты по тексту (с образцами)
- Слои Optional Content
- Признаки OCG в сыром PDF

### 3.5. `compare_pdf_metadata.py`

**Назначение:** Полное сравнение двух PDF: метаданные, XMP, шрифты, структура.

**Использование:**
```
python3 compare_pdf_metadata.py "input ru.pdf" "Квитанция (21).pdf"
```

**Выводит:**
- JSON-дамп обоих файлов
- Diff метаданных
- Сравнение шрифтов

### 3.6. Обёртки запуска

**`rebuild_квитанция.py`** — вызывает `rebuild_pdf.py` с:
- Генерацией случайного номера счёта (4081 + 16 цифр)
- Типовыми заменами (счёт, номер операции, сумма, телефон, банк, ФИО, даты, идентификатор СБП)
- Опциями `--strict-forensic`, `--font-alias-from-input`

**`run_rebuild.py`** — аналогично, с фиксированным путём к `input ru.pdf`, `donor.pdf`.

**`copy_квитанция_11.py`** — пример точечной замены 200→100 через `content_stream_replace.py`.

## 4. Справочник Альфа-Банка (`ALFA_BANK_RECEIPT_REFERENCE.md`)

Справочник содержит:

1. **Текстовые маркеры** — «АО «АЛЬФА-БАНК»», «Выписка по счету», «Квитанция о переводе по СБП» и др.
2. **Фирменный стиль** — цвета (чёрный, красный, белый), шрифты (Styrene A, Helvetica).
3. **Структура PDF** — Producer iOS Quartz, версия 1.3, размеры шрифтов (8/9/12/18 pt).
4. **Типичные поля** — выписка, квитанция СБП, перевод за рубеж.
5. **Формат номера операции** — C + 15 цифр (СБП) или C + 14 цифр (международный).
6. **Код банка в СБП** — 100000000008.
7. **Эвристики определения банка** — веса признаков.
8. **Рекомендуемый алгоритм проверки** — извлечение текста → маркеры → структура.

## 5. Процесс пересборки (Clean Room)

### 5.1. Этапы

1. **Открытие input PDF** — чтение через PyMuPDF (fitz).
2. **Для каждой страницы:**
   - Создание новой пустой страницы того же размера.
   - Вставка шрифтов (Tahoma/Arial или из исходного).
   - Копирование изображений по xref (без перекодирования).
   - Копирование векторных объектов (drawings).
   - Извлечение текста (get_text "dict").
   - Для каждого блока/line/span: применение замен, отрисовка с учётом стиля.
3. **Применение замен** — через `replace_text()` с токен-границами (alnum, кириллица).
4. **Числовые поля** — при `--numbers-grow-left` правая граница фиксируется, текст растёт влево.
5. **Сохранение** — garbage=4, clean=True, deflate (если optimize), subset fonts.
6. **Пост-обработка** (при `--native-structure`):
   - fzImg0/1/2 → Im1/2/3 в content streams.
   - forced_tahoma/unicode_fallback → G1.
   - Producer MuPDF → iOS Quartz.
   - Удаление/нейтрализация Metadata.
   - При `--ios-match`: BaseFont и Document ID из эталона.

### 5.2. Forensic-рациональность

- Страницы не копируются бинарно.
- Новое тело файла генерируется с нуля.
- `save(..., garbage=4, clean=True)` переписывает xref.
- Проверка `forensic_self_check`: startxref, %%EOF, подозрительные объекты, отпечатки редакторов.

## 6. Проверка результата (forensic)

Инструкции из `pdf_verification_instructions.txt`:

1. **qpdf --check output.pdf** — структурная валидность.
2. **startxref=1, %%EOF=1** — признак «чистой» перезаписи.
3. **Метаданные** — producer/creator без iLovePDF, Canva, Photoshop.
4. **Подозрительные объекты** — /JavaScript, /JS, /OpenAction и т.д.
5. **Editor fingerprints** — photoshop, canva, ilovepdf и т.д.
6. **Цифровые подписи** — pdfsig input.pdf.

## 8. Типовые сценарии использования

### 8.1. Полная пересборка с заменами

```bash
python3 rebuild_pdf.py input.pdf output.pdf \
  --donor-pdf donor.pdf \
  --strict-forensic \
  --font-alias-from-input \
  --replace "40817810980480002476=НОВЫЙ_СЧЁТ" \
  --replace "10=5 000" \
  --replace "ВТБ=Озон Банк (Ozon)"
```

### 8.2. Точечная замена одной суммы

```bash
python3 content_stream_replace.py "Квитанция 11.pdf" "Квитанция 11_copy.pdf" \
  --replace "200=100" --preserve-id
```

### 8.3. Замена из JSON

```json
{"10": "1000", "22.02.2026 17:49": "24.02.2026 12:20"}
```

```bash
python3 rebuild_pdf.py input.pdf output.pdf --replacements-json replacements.json
```

### 8.4. Совпадение с iOS-эталоном

```bash
python3 rebuild_pdf.py input.pdf output.pdf --ios-match --replace "OLD=NEW"
```

## 9. Ограничения и особенности

1. **Шрифты Type0/Identity-H** — при пересборке могут заменяться на Tahoma с алиасом из input.
2. **Разделение текста по span'ам** — при замене значения, разбитого на несколько span'ов, используется line-level replacement (отключён по умолчанию из-за риска нарушения табличной вёрстки).
3. **Метаданные** — при `--copy-input-metadata` копируются как есть; при `--native-structure` Metadata stream нейтрализуется.
4. **Document ID** — при `--ios-match` берётся из эталона.
5. **Изменение сумм** — рекомендуется `--numbers-grow-left` для полей с RUR/TJS/UZS.

## 10. Глоссарий

| Термин | Описание |
|-------|----------|
| Clean Room | Подход: перерисовка всех элементов на новом документе, без копирования страниц |
| Donor PDF | PDF-источник для копирования стиля текста (шрифт, размер, цвет) |
| Forensic | Критерии отсутствия следов редактирования |
| fzImg / Im | Имена ресурсов изображений в content stream (PyMuPDF vs iOS) |
| BaseFont | Имя шрифта в ресурсах PDF (например, AAAAAB+font000000002f4db77f) |
| startxref / %%EOF | Маркеры структуры PDF; >1 указывает на инкрементальное обновление |
| СБП | Система быстрых платежей |
| Redaction | Редактирование путём закрашивания и наложения нового текста |
