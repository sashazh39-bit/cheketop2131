## 2. Фирменный стиль Альфа-Банка

### Цвета
- **Основной**: чёрный (#000000)
- **Акцент**: красный
- **Фон**: белый

### Шрифты
- **Бренд**: Styrene A (маркетинг, 2019+)
- **В PDF**: встроенный CID-шрифт (font000000002f4e6404 или аналогичный хэш)
- **Дополнительно**: Helvetica для дат (напр. «02.03»)

### Логотип
- Литера «А» с подчёркиванием
- Красная капля вместо точки над «i» в слове «Альфа»

## 3. Структура PDF (по анализу input2.pdf)

### Метаданные
- **Producer**: `iOS Version X.X.X Quartz PDFContext` (при экспорте из приложения)
- **PDF Version**: 1.3
- **Font count**: 3–4 (выписка); 1 (квитанция СБП)

### Размеры шрифтов
| Элемент | Размер (pt) | Шрифт |
|---------|-------------|-------|
| Заголовок страницы («Выписка по счету») | 18 | font* |
| Секция («Операции по счету») | 12 | font* |
| Подпись/уполномоченное лицо | 9 | font* |
| Основной текст | 8 | font* |
| Подписи к полям | 8 | font* |

### Отпечаток шрифтов (input2.pdf)
- **Встроенный**: `font000000002f4e6404` (CIDFontType2, Identity-H) — iOS заменяет системный шрифт на общий хэш
- **Helvetica**: `OCVIBV+Helvetica` — для фрагментов дат
- Оригинальное имя шрифта (Styrene A) при экспорте через iOS не сохраняется

### Типичные поля выписки
- Номер счета
- Дата открытия счета
- Валюта счета (RUR)
- Тип счёта (Текущий счёт)
- Клиент, адрес регистрации
- Входящий/исходящий остаток
- Платежный лимит
- Дата проводки, Код операции, Описание, Сумма

### Формат номера операции
- **СБП**: `C` + 15 цифр (напр. `C422402260356995`, `C160303262050347`)
- **Международный перевод**: `C` + 14 цифр (напр. `C822202260011245`, `C822502260006543`)
- Первые 6 цифр после C: дата (YYMMDD)

## 4. Структура квитанции СБП (общая для банков)

Стандартный набор полей:
```
Сформирована [дата] мск
Квитанция о переводе по СБП
Сумма перевода / Комиссия / Дата и время
Номер операции / Получатель / Номер телефона
Банк получателя / Счёт списания / Идентификатор операции в СБП
Сообщение получателю
```

**Отличие Альфа-Банка** — в явном виде «банк отправителя» в СБП-квитанции не указывается.  
Идентификатор по внешнему виду и метаданным PDF при экспорте через iOS.

## 7. Эвристики для определения «это Альфа-Банк»

| Признак | Вес | Описание |
|---------|-----|----------|
| Текст «АО «АЛЬФА-БАНК»» | 100% | Прямое упоминание |
| Текст «АЛЬФА-БАНК» | 95% | Маркер бренда |
| «Выписка по счету» + характерная структура | 80% | Типичная выписка Альфа-Банка |
| Счёт 40817810980480002476 | 0% | Сам по себе банк не определяет |
| Шрифты 8/9/12/18 pt + Helvetica для дат | 60% | Совпадение со структурой выписки |
| Producer: iOS Quartz | 0% | Общий для всех банков при экспорте из iOS |

---

## 8. Что НЕ является надёжным признаком

- **Номер счёта** — без БИК банк по счёту однозначно не определяется
- **Producer PDF** — при сохранении через iOS общий для всех банков
- **Идентификатор СБП** — публичной расшифровки формата нет
- **Шрифты** — при экспорте в PDF часто заменяются на встроенные CID

---

## 9. Рекомендуемый алгоритм проверки

```
1. Извлечь текст из PDF (OCR или get_text)
2. Если найдено «АО «АЛЬФА-БАНК»» или «АЛЬФА-БАНК» → банк Альфа
3. Если «Выписка по счету» + структура полей выписки → сравнить с эталоном
4. Для СБП-квитанций: сравнить шрифты, размеры, порядок полей с эталоном Альфа-Банка
5. При необходимости — проверка через StopScam API (sender_bank_full_name)
```
