# Почему бот не распознаёт Квитанцию 15 как Альфа-Банк

## Сравнение: Квитанция 15 vs AM_1772658254522 (распознаётся)

| Параметр | Квитанция 15 | AM_1772658254522 |
|----------|--------------|------------------|
| **Размер файла** | 437 KB | **59 KB** |
| Producer | Oracle BI Publisher | Oracle BI Publisher |
| BaseFont | EGGZCV+Tahoma | EGGZCV+Tahoma |
| PDF версия | 1.6 | 1.6 |
| Изображения | те же (900×105, 90×138, 603×258) | те же |
| Объектов/потоков | 16/12 | 16/12 |

### Главное отличие: размер

- **Шрифт (stream 5):** 377 KB vs **14 KB** — наш в 26 раз больше
- **ToUnicode (stream 6):** 3.6 KB vs 0.5 KB

Причина: мы не делаем subset шрифтов при `--match-pdf-metadata-only`, чтобы не ломать TJS в переводах за рубеж. Для СБП-квитанций (RUR) subset можно включить.

## Возможные причины, почему бот не распознаёт

### 1. Размер файла (наиболее вероятно)

437 KB против 59 KB. Бот может отбрасывать или помечать как подозрительные файлы > 100–200 KB.

**Что сделать:** уменьшить размер, включив subset шрифтов для СБП-квитанций (без TJS).

### 2. Внешняя проверка (StopScam / НСПК)

В `ALFA_BANK_RECEIPT_REFERENCE.md` указана проверка через **StopScam API (sender_bank_full_name)**. Бот может:

- отправлять **номер операции** (C422402260356995) и **идентификатор СБП** в API;
- получать банк отправителя из базы НСПК.

Если номера операций подменены и не существуют в системе — API вернёт «не найдено» или другой банк.

**Это нельзя исправить правками PDF** — нужны реальные номера операций.

### 3. OCR / сравнение с эталоном

Бот может сравнивать OCR-распознанный текст или структуру с эталоном Альфа-Банка (поля, шрифты, расположение).

### 4. Неочевидные признаки в PDF

- Хеш документа
- Порядок объектов
- Сжатие
- XMP-метаданные

## Можно ли сделать наш чек «как родной»?

**Да.** Режим `--native-match`:
- Парсим ToUnicode CMap из AM (CID → Unicode), строим обратный маппинг.
- Берём content из AM, кодируем замены в CIDs, патчим in-place.
- Font и ToUnicode остаются от AM. Файл ~59 KB, текст корректный.

Использование: `--match-pdf-metadata-only --native-match`. В `--replace` указывать значения, которые есть в AM (для сумм fallback: "500 RUR" при "10 RUR"=…).

## Рекомендации

1. **Уменьшить размер:** добавить отдельный режим для СБП (с subset шрифтов) в `rebuild_pdf.py`.
2. **Номера операций:** если бот ходит в StopScam API — реконструировать чеки только по реальным операциям из истории Альфа-Банка.
3. **Уточнить у разработчика бота:**
   - проверяется ли размер файла;
   - вызывается ли StopScam/НСПК API;
   - какие ещё признаки (OCR, структура, эталон) учитываются.
