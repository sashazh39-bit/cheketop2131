# Команды и инструкция по сборке квитанций

**Дата:** март 2026  
**Проект:** чекетоп — пересборка PDF-квитанций с forensic-совпадением

---

## 1. Основная команда (полное совпадение с iOS-эталоном)

```bash
python3 rebuild_pdf.py "input ru.pdf" "Квитанция 8.pdf" \
  --ios-match \
  --replace "10=100" \
  --replace "Александр=Иван"
```

**Что делает:**
- Пересобирает PDF из input (изображения + текст на новом документе)
- `--ios-match` — полное совпадение с эталоном: Producer, BaseFont, Document ID, структура (Im1/Im2/Im3, G1), без MuPDF
- Заменяет текст: «10»→«100», «Александр»→«Иван»

---

## 2. Команда с указанием даты метаданных

Дата в метаданных (creationDate, modDate) влияет на forensic-проверку. Её нужно выставлять близко к дате операции в тексте чека.

```bash
python3 rebuild_pdf.py "input ru.pdf" "Квитанция 8.pdf" \
  --ios-match \
  --creation-date "2026-02-24 10:40" \
  --replace "10=100" \
  --replace "Александр=Иван"
```

**Формат даты:** `YYYY-MM-DD HH:MM`, `YYYY-MM-DD HH:MM:SS` или `DD.MM.YYYY HH:MM`  
Примеры: `2026-02-24 10:40`, `2026-02-24 14:30:15`, `24.02.2026 10:40`

**Отдельная дата изменения:** `--mod-date "2026-02-24 15:00"` — если не задана, используется creation-date.

## 3. Без замен текста

```bash
python3 rebuild_pdf.py "input ru.pdf" "Квитанция 8.pdf" --ios-match
```

## 4. Режим Oracle/AM (--match-pdf)

Подстройка под эталон Oracle BI Publisher (AM_1772641049914.pdf): метаданные, Producer, BaseFont, Document ID, версия PDF 1.6.

```bash
python3 rebuild_pdf.py "input ru.pdf" "output.pdf" \
  --match-pdf "AM_1772641049914.pdf" \
  --replace "10=150" \
  --replace "Александр=Демид"
```

**Что копируется из эталона:**
- Producer (Oracle BI Publisher 12.2.1.4.0)
- Document ID
- BaseFont (KKGMOS+Tahoma)
- Версия PDF (1.6)
- creationDate/modDate (если есть в эталоне)

**Рекомендация:** использовать AM как input — так PDF не повреждается:
```bash
# AM как источник и эталон — надёжный вариант (полное совпадение)
python3 rebuild_pdf.py "AM_1772641049914.pdf" "Квитанция222.pdf" \
  --match-pdf "AM_1772641049914.pdf" \
  --match-pdf-metadata-only
```

**Что делает `--match-pdf-metadata-only`:**
- Producer: только Oracle (MuPDF убирается)
- Document ID: копируется из эталона
- BaseFont: Tahoma вместо Tahoma Regular
- Версия PDF 1.6, даты из эталона

**С заменами текста:**
```bash
python3 rebuild_pdf.py "AM_1772641049914.pdf" "Квитанция.pdf" \
  --match-pdf "AM_1772641049914.pdf" \
  --match-pdf-metadata-only \
  --replace "150=200" \
  --replace "Демид=Иван" \
  --creation-date "2026-03-04 19:22"
```

## 5. С другим input (эталоном)

Свойства выходного PDF (Producer, BaseFont, Document ID) берутся из первого аргумента (input):

```bash
# Как iOS-квитанция (input ru.pdf)
python3 rebuild_pdf.py "input ru.pdf" "output.pdf" --ios-match

# Как Oracle-квитанция
python3 rebuild_pdf.py "20251021220717 (1).pdf" "output.pdf" --ios-match
```

## 5. Расширенные замены из JSON

```bash
python3 rebuild_pdf.py "input ru.pdf" "output.pdf" \
  --ios-match \
  --replacements-json replacements.json
```

Формат `replacements.json`:
```json
{
  "10": "100",
  "Александр": "Иван",
  "24.02.2026": "25.02.2026"
}
```

## 6. Команды без --ios-match (минимальная пересборка)

## 7. Все опции rebuild_pdf.py

| Опция | Описание |
|-------|----------|
| `input_pdf` | Исходный PDF |
| `output_pdf` | Выходной PDF |
| `--replace OLD=NEW` | Замена текста (можно повторять) |
| `--replacements-json FILE` | JSON с заменами |
| `--creation-date "YYYY-MM-DD HH:MM"` | Дата создания в метаданных |
| `--mod-date "YYYY-MM-DD HH:MM"` | Дата изменения (если не задана = creation-date) |
| `--ios-match` | Эталон iOS: BaseFont, Producer, Document ID из input |
| `--match-pdf PATH` | Эталон Oracle/AM: метаданные, Producer, Document ID, версия PDF из указанного файла |
| `--match-pdf-metadata-only` | С --match-pdf: лёгкий патч без native-structure; копирует Producer, Document ID, BaseFont Tahoma |
| `--native-structure` | Im1/Im2/Im3, G1, без Metadata |
| `--patch-header` | Убрать MuPDF из заголовка |
| `--donor-pdf FILE` | PDF для копирования стиля текста |
| `--donor-text "..."` | Текст для поиска стиля в donor |
| `--no-copy-metadata` | Не копировать метаданные из input |
| `--no-preserve-source-fonts` | Использовать Tahoma вместо исходных шрифтов |

## 8. Метаданные — когда менять дату

| Поле | Что означает | Когда менять |
|------|--------------|--------------|
| **creationDate** | Дата создания PDF | Близко к дате операции в чеке |
| **modDate** | Дата последней модификации | Обычно = creationDate |
| **producer** | Программа создания | Копируется из input |
| **creator** | Создатель контента | Часто пусто |

**Рекомендация:** Задавать `--creation-date` по дате и времени перевода в тексте чека. Если чек за 24.02.2026 14:30 — используйте `--creation-date "2026-02-24 14:30"`.

**Document ID** (с `--match-pdf-metadata-only`): автоматически копируется из эталона. Варианты:
1. **Одинаковый у всех** — одна серия квитанций с одним ID (как у эталона)
2. **Уникальный** — нужен отдельный скрипт post-processing с генерацией нового ID
3. **По умолчанию** — копирование из эталона, forensic-совпадение

# С датой метаданных = дате операции в чеке

python3 rebuild_pdf.py "input ru.pdf" "Квитанция 8.pdf" --ios-match
