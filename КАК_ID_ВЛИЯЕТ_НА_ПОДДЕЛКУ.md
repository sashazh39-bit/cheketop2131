# Как замена Document ID влияет на определение подделки

## Где находится /ID в PDF

`/ID` — это поле в **trailer** (конец файла):

```
trailer
<<
/Root 1 0 R
/Info 3 0 R
/ID [<8C7A21FFE15152289E6AEC6F80509DAF> <8C7A21FFE15152289E6AEC6F80509DAF>]
/Size 19
>>
startxref
8882
%%EOF
```

- **Позиция:** последние ~100–150 байт файла
- **Формат:** два 32-символьных hex через пробел
- **Размер блока:** ~69 байт

## Что меняется при замене /ID

| Параметр | До замены | После замены |
|----------|-----------|--------------|
| Длина файла | не меняется (подмена in-place) |
| SHA256 всего файла | **меняется** |
| Content streams | не трогаются |
| xref, startxref | не трогаются |

Заменяются только 69 байт в trailer, но этого достаточно, чтобы файл считался другим.

## Почему бот помечает чек как подделку

### 1. Хеш файла

Любое изменение (в т.ч. /ID) меняет SHA256/MD5 файла. Если бот хранит хеши легитимных чеков, модифицированный файл в базу не попадёт и будет считаться подделкой.

### 2. Проверка /ID

- Банк может генерировать /ID по своей схеме (например, из timestamp + части контента).
- Наш `update_id` использует `MD5(весь_файл)` — другой принцип формирования.
- Возможна проверка, что /ID принадлежит определённому банку или типу чека.

### 3. Согласованность полей

- Часто `/ID[0] == /ID[1]` у оригинальных PDF.
- `/CreationDate` может быть связан с /ID (например, оба зависят от timestamp).
- При замене только /ID эти связи рвутся, что может вызывать подозрения.

### 4. База известных чеков

Бот может сверять /ID с базой уже загруженных или выданных банком чеков. Новый /ID там не будет найден → «подделка».

## Рекомендация

**Не менять /ID** при патче — вызывать `patch_from_values` с `keep_metadata=True`:

```python
out = patch_from_values(
    data, donor,
    ...,
    keep_metadata=True,  # /ID и CreationDate остаются от донора
)
```

Минус: при повторном использовании одного и того же донора несколько чеков получат одинаковый /ID; бот может это обнаружить. В таком случае нужны либо разные доноры, либо другой способ верификации (не через /ID).
