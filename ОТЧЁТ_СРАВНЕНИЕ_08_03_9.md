# Полное сравнение: оригинал vs пропатченный `08-03-26_00-00 9.pdf`

## Задача
- Замена плательщика: **Александр Евгеньевич Ж.** → **Евгений Александрович Е.**
- Замена суммы: **10 ₽** → **10 000 ₽**
- Результат сохранён в `VEGA08.03/08-03-26_00-00 9.pdf`

---

## 1. Размеры файлов

| Файл | Размер |
|------|--------|
| Оригинал | 9 268 байт |
| Пропатченный | 9 275 байт |
| **Разница** | **+7 байт** |

Увеличение связано с ростом сжатого потока (длиннее TJ-блоки для плательщика и суммы).

---

## 2. Изменения в декомпрессированном content stream

### 2.1 Сумма (10 ₽ → 10 000 ₽)

**Оригинал (CID-кодировка):**
```
(\x00\x14)-11.11111 (\x00\x13)-11.11111 (\x00\x03)-11.11111 (\x04@)
```
- `0014` = "1"
- `0013` = "0"
- `0003` = пробел
- `04@` = "₽"
- Текст: **10 ₽**

**Пропатченный:**
```
(\x00\x14)-11.11111 (\x00\x03)-11.11111 (\x00\x13)-11.11111 (\x00\x13)-11.11111 (\x00\x13)-11.11111 (\x00\x03)-11.11111 (\x04@)
```
- Текст: **10 000 ₽**

Tm (матрица трансформации) для суммы: `231.52501 72.37499` → `195 72.37499` (сдвиг влево для размещения 6 символов вместо 4).

### 2.2 Плательщик

**Оригинал (CID):**
```
(\x02\x1c)-16.66667 (\x02G)-16.66667 (\x02A)-16.66667 (\x02F)-16.66667 (\x02M)-16.66667 (\x02<)-16.66667 (\x02I)-16.66667 (\x02@)-16.66667 (\x02L)-16.66667 ...
```
→ Текст: **Александр Евгеньевич Ж.**

**Пропатченный:**
```
(\x02\x21)-16.66667 (\x02>)-16.66667 (\x02?)-16.66667 (\x02A)-16.66667 (\x02I)-16.66667 (\x02\x24)-16.66667 ...
```
→ Текст: **Евгений Александрович Е.**

Tm для плательщика: сохранён/скорректирован для выравнивания.

## 3. Структурные отличия

| Компонент | Изменения |
|-----------|-----------|
| **/Length** | Значение потока: увеличено на ~14 байт (длина сжатых данных) |
| **xref** | Смещения объектов после потока сдвинуты на +7 байт |
| **startxref** | Скорректирован при необходимости |
| **%PDF** / версия | Без изменений |
| **Словарь потока** | Без изменений (тип, фильтр) |
| **Сжатие** | zlib level 6 (как в оригинале) |
| **/ID** | Сохранён |

---

## 4. Сводка отличий

1. **Текст плательщика** — заменён ФИО полностью.
2. **Текст суммы** — заменён с «10 ₽» на «10 000 ₽».
3. **Tm для суммы** — уменьшен X-координата (231.5 → 195) для размещения более длинной надписи.
4. **Размер потока** — +7 байт после пересжатия.
5. **xref/startxref** — обновлены под новый размер потока.

QR-код и фискальные данные в PDF не изменяются — они находятся вне редактируемого content stream. Визуально чек отображает «Евгений Александрович Е.» и «10 000 ₽» вместо исходных значений.
