# Почему чек «подделан» при прохождении целостности

## Ситуация

- **чек_15_03_иван.pdf**: целостность ✓, но верификатор пишет «подделан»
- **чек_15_03_максим.pdf**: не проходит ничего

## Причина «подделан» (из СРАВНЕНИЕ_ЧИСТЫЙ_VS_ВАЛИДНЫЙ)

Верификатор отправляет в API (StopScam / НСПК) **operation_id** и проверяет согласованность:
- Для operation_id в системе хранятся: сумма, счёт, телефон, ФИО, дата
- Если в чеке другие значения → **«подделан»**

Мы берём **operation_id из шаблона 15-03-26**, но меняем:
- ФИО → «Иван Петрович О.» (не из 15-03-26)
- Сумму → 5000 (не из 15-03-26)
- Телефон → «+7 (916) 123-45-67» (не из 15-03-26)

**Итог:** operation_id соответствует одной операции, а содержимое — другой. API считает это подделкой.

## Дополнительные факторы

1. **Телефон** — жёстко связан с operation_id. Случайный или новый номер почти всегда даёт «подделан».
2. **Document ID** — изменение даже одного символа может учитываться при проверке.
3. **Метаданные** — Producer, Creator, CreationDate: любое несоответствие может учитываться.

## Что можно пробовать (эксперимент)

| Опция | Суть | Команда |
|-------|------|---------|
| `--keep-phone` | Оставить телефон из 15-03-26 | `python3 gen_receipt_15_03.py out.pdf --keep-phone --payer "Иван Петрович О." --amount 5000` |
| `--keep-id` | Не менять Document ID (оставить как в шаблоне) | `python3 gen_receipt_15_03.py out.pdf --keep-id ...` |

## Ограничение

Чтобы пройти проверку «подделан», **все поля** должны совпадать с записью в системе для данного operation_id. То есть:
- operation_id, телефон, сумма, ФИО, банк, счёт, дата — из одной реальной операции.

Использовать «свои» ФИО и сумму можно только если есть реальный чек с этим operation_id и этими же данными. Смешивать operation_id одной операции и данные другой нельзя.

## Рекомендация

Для чека с **твоими** данными (ФИО, сумма, телефон) нужен **реальный СБП-перевод** с этими данными. Тогда:
1. Есть настоящий PDF от банка → лучше использовать его.
2. Либо operation_id из этого перевода, и в чеке — точно такие же ФИО, сумма, телефон, банк, счёт, дата.
