# Правила проверки чеков (НЕ МЕНЯТЬ)

**Зафиксировано:** чек проходит проверку бота полностью, если в нём изменяются **только**:

1. **Дата и время**
2. **ФИО отправителя и получателя**
3. **Сумма**
4. **Номер телефона**
5. **Document ID** — только **один символ** (в hex-строке /ID)

## Критично

- **Document ID:** менять разрешено **строго один символ** в hex. Полная замена /ID → бот помечает подделку.
- **CreationDate:** должен совпадать с датой/временем в чеке (после патча вызывать `update_creation_date` с этой датой).
- **Метаданные:** при патче использовать `keep_metadata=True`, затем точечно менять 1 символ в /ID.

## Чего не менять при «проходящем» чеке

- operation_id (ID операции) — оставить от донора
- банк, счёт — оставить или менять осторожно
- Producer, Creator и др. — не трогать

## Генерация «проходящих» чеков

Использовать **gen_verified_receipt.py** — применяет только разрешённые изменения и проверенных доноров:

```bash
python3 gen_verified_receipt.py output.pdf --payer "ФИО получателя" --recipient "ФИО отправителя" --amount 14600 --phone "+7 (992) 494-94-95"
```
