# Сравнение check_21, check_11 и receipt_verified_test

## Итог: почему чеки не проходят

**Разный донор → разная структура → бот отклоняет** (см. БОТ_ОТКЛОНЯЕТ_ЧЕК.md)

| Файл | Донор | BaseFont | /W hash |
|------|-------|----------|---------|
| **check_21.pdf** | 21-04-25_23-38.pdf | AAPHLC+SFProDisplay-Regular | 29b2d020... |
| **check_11.pdf** | Станислав Геннадьевич З..pdf | AANJPC+SFProDisplay-Regular | febf2e81... |
| **receipt_verified_test.pdf** | Андрей Викторович С..pdf | AAIQLC+SFProDisplay-Regular | e4774628... |

У каждого донора свой BaseFont subset, свой /W (ширины глифов), свои content streams. Бот, судя по всему, сравнивает структуру с эталоном или базой «легитимных» чеков.

## Общее у всех трёх

- Producer: openhtmltopdf.com ✓
- CreationDate: обновлён под дату в чеке ✓
- Document /ID: изменён 1 символ ✓
- keep_metadata=True при патче ✓

## Различия

### 1. Структура PDF

- Размер: check_21 (9337), check_11 (9410), receipt_verified (9130) — разные
- Content streams: хеши не совпадают, разный контент
- ToUnicode, /W — разные у каждого донора

### 2. Метаданные

| | check_21 | check_11 | receipt_verified |
|---|----------|----------|------------------|
| CreationDate | D:20260313132811+03'00' | D:20260313132811+03'00' | D:20260313131700+03'00' |
| /ID | E5D32...4A | A6CA4...0B | 75FBA...98 |

## Рекомендация

**Если receipt_verified_test проходит** — бот, вероятно, знает структуру донора Андрей Викторович С. (AAIQLC). Нужно всегда использовать этот донор:

```python
DONOR = "база_чеков/vtb/СБП/Андрей Викторович С..pdf"
```

**Если все три не проходят** — причина может быть в:

1. Смене даже одного символа в /ID (бот видит изменение)
2. CreationDate: в receipt_verified секунды = 00; в check_21/11 — фактические (13:28:11)
3. Других проверках (хеш файла, признаки правки и т.п.)
