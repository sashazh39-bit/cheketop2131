# Анализ: где в PDF встречается номер счёта

## Результаты проверки AM_1772740244320.pdf

### 1. Номер счёта в документе

| Место | Найден? |
|-------|---------|
| Content stream (текст страницы) | ✓ Единственное вхождение |
| Метаданные (/Info, Producer, Title и т.д.) | ✗ Нет |
| XMP | ✗ Не используется |
| Аннотации | ✗ Нет аннотаций |
| Поля форм (widgets) | ✗ Нет полей |
| Цифровая подпись | ✗ Отсутствует |
| Сырой ASCII в файле | ✗ Счёт в CID (сжатый hex), не в открытом виде |

**Вывод:** номер счёта есть только в **content stream**, в виде CID-hex. Дубликатов и скрытых копий нет.

---

### 2. Структура объектов

- **Obj 1** — Catalog
- **Obj 2** — Info (Producer)
- **Obj 3** — Pages
- **Obj 4** — Resources (шрифты, изображения)
- **Obj 5–7** — изображения
- **Obj 8** — Page
- **Obj 9** — Contents (единственный content stream с текстом)
- **Obj 10–12** — шрифт
- **Obj 13–14** — ToUnicode, embedded font
- **Obj 15–16** — destinations для Outline (закладки, координаты)

Скрытых или неочевидных объектов с номером счёта нет.

---

### 3. Сравнение оригинал vs receipt_account_replaced.pdf

| Stream | Содержимое | Оригинал vs Заменённый |
|--------|------------|-------------------------|
| 0 | Картинка | Идентичны |
| 1 | Картинка | Идентичны |
| 2 | Картинка | Идентичны |
| 3 | Content (текст) | Отличаются только CID-блок счёта (~17 байт) |
| 4 | ToUnicode/шрифт | Идентичны |
| 5 | Шрифт | Идентичны |

В PDF меняется только content stream: один CID-блок (номера счёта).

---

## Вывод

- Номер счёта нигде больше не дублируется.
- Цифровой подписи нет.
- Структура и остальные объекты совпадают.
- Различие — только CID-последовательность цифр счёта в content stream.

Значит, бот не может опираться на вторую копию счёта, подпись или другую скрытую структуру. С большой вероятностью проверка идёт по содержимому или представлению именно **блока счёта в content stream** (хеш фрагмента, паттерн CID и т.п.).
