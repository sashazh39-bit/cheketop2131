# Анализ: верификация vs целостность

## Текущая ситуация

| Шаблон | Метод | Верификация | Целостность |
|--------|-------|-------------|-------------|
| 16-03-26 | add_glyphs (Максим/Анна) | ✅ | ❌ (инъекция М,Н,Р) |
| 02-03-26 | gen_verified (content-only) | ❌ | ✅ |

## Ключевые отличия структур

### 02-03-26 vs 16-03-26

| Параметр | 02-03-26 (целостность OK) | 16-03-26 (верификация OK) |
|----------|---------------------------|---------------------------|
| **BaseFont** | AATKOC+SFProDisplay-Regular | AAMQMC+SFProDisplay-Regular |
| **/W массив** | 375 bytes, hash b6ef0d52... | 356 bytes, hash 2c07416... |
| **ToUnicode** | obj 17: 3703→5036 bytes | obj 17: 3768→5184 bytes |
| **Content streams** | Другая структура | Другая структура |
| **Document ID** | F5CB8C58... (02-03-26) | 1F86B305... (16-03-26) |
| **CreationDate** | из патча ||

### Document ID по шаблон из оригинала ам
- **16-03-26_00-00.pdf**: 1F86B305E880902507C91BA4F2133DF1
- **16-03-26_00-00 2.pdf**: 07AC8810705EB5436F4200CCE0137882
- **02-03-26_00-47.pdf**: F5CB8C5879D7F2120B7A8FB7127F150D

## Гипотезы почему верификация не проходит (02-03-26)

1. **Document ID** — верификатор может проверять ID против белого списка «оригинальных» 16-03-26 серии
2. **BaseFont** — ожидается AAMQMC, а не AATKOC
3. **Структура целиком** — хеш/подпись привязаны к 16-03-26 формату

## Варианты решения

### 1. Подмена только Document ID (минимальное изменение)
Чек с 02-03-26 (целостность ✅), заменить /ID на ID из 16-03-26_00-00 2.pdf (с изменением 1 символа).
- Целостность: не трогаем
- Верификация: проверяем, достаточно ли ID из «правильной» серии

### 2. Копирование метаданных
Скопировать из 16-03-26_00-00 2.pdf: ID, CreationDate, Producer.
- Сохраняем структуру 02-03-26, меняем только метаданные

### 3. Поиск шаблона с AAMQMC и полным алфавитом
Шаблон с тем же BaseFont что 16-03-26, но со всеми буквами → content-only + Максим/Анна.

### 4. Трансплантация структуры (сложно)
Взять 16-03-26 как базу, заменить только content streams из 02-03-26 — очень трудоёмко и рискованно.

## Рекомендуемый порядок проверки

1. Вариант 1: только ID из 16-03-26 2.pdf (+1 символ)
2. Вариант 2: ID + CreationDate + Producer из 16-03-26 2.pdf
