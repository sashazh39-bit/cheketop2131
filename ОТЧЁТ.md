# Функциональный отчёт: Telegram-бот для работы с PDF-чеками

**Дата:** 21.03.2026  
**Проект:** `/Desktop/чекетоп`  
**Файлы бота:** `bot_standalone.py` (основной), `bot.py` (вспомогательный)

---

## 1. Назначение

Бот предназначен для **модификации PDF-чеков банков (Альфа-Банк, ВТБ)** на уровне бинарных данных. Поддерживает замену сумм, дат, ФИО, телефонов, номеров счетов, а также генерацию чеков из базы донорских PDF и создание банковских выписок. Все изменения вносятся на уровне CID-шрифтов и xref-таблиц без использования PDF-библиотек, что делает правки сложноотличимыми от оригинала.

## 4. Режимы работы

### 4.1 Патчинг загруженного чека (`📄 Загрузить чек`)

Пользователь загружает PDF → бот предлагает выбор банка:

| Кнопка | Режим |
|---|---|
| **Альфа-Банк** | Замена суммы (формат `ОТКУДА КУДА`) |
| **ВТБ** | Замена суммы (формат `ОТКУДА КУДА`) |
| **ВТБ — все поля** | 6-шаговый мастер: сумма, дата, ФИО плательщика, ФИО получателя, телефон, банк |
| **Альфа СБП — все поля** | 6-шаговый мастер: сумма, дата/время, получатель, телефон, банк, последние 4 цифры счёта |
| **ВТБ Трансгран** | Международные переводы ВТБ (сумма, курс, зачислено, ФИО, телефон, счёт, дата) |
| **Альфа Трансгран** | Международные переводы Альфа (сумма, комиссия, курс, зачислено, телефон, ФИО, ID операции) |
| **Авто** | Автоопределение банка |

**Шаговые мастера** поддерживают кнопку «оставить текущее» на каждом шаге — бот автоматически извлекает текущие значения из PDF и предлагает их по умолчанию.

---

### 4.2 Генерация чека из базы (`✨ Сгенерировать`)

Бот выбирает донорский чек из `база_чеков/vtb/СБП/` и собирает новый чек по введённым параметрам.

**Шаги:**
1. ФИО плательщика
2. ФИО получателя
3. Сумма
4. Дата
5. Банк получателя
6. Последние 4 цифры счёта

**Три пути генерации:**
- `run_sbp_generate_15_03()` — шаблон `15-03-26_00-00.pdf` + `patch_from_values()`
- `run_sbp_generate_verified()` — донор `check(3).pdf`, только контент (без модификации шрифта)
- `run_sbp_generate()` — subprocess-вызов `add_glyphs_to_13_03.py` для инъекции глифов

После генерации можно создать заявку по этому чеку.

---

### 4.3 Создание выписки (`📋 Создать выписку`)

Два подрежима:

**a) Редактирование выписки (`stmt_edit`):**
1. Загрузить PDF выписки
2. Ввести пары замен (строки `СТАРОЕ=НОВОЕ`)
3. Опционально: заменить ФИО, баланс, телефон, ID заявки
4. Получить пропатченный PDF

**b) Генерация из чека (`stmt_receipt`):**
1. Загрузить PDF чека (бот извлекает сумму, дату, ФИО)
2. Ввести начальный баланс
3. Бот патчит шаблон `база_выписок/vyписка_один_платеж.pdf`
4. Получить готовую выписку

---

### 4.4 База доноров (`📂 Проверка базы`)

| Функция | Описание |
|---|---|
| Просмотр статистики | Количество ВТБ СБП / ВТБ-на-ВТБ / Альфа чеков в индексе |
| Добавить в индекс | Загрузить новый PDF → бот добавляет его в нужную папку и индекс |
| Пересобрать индекс | Полное сканирование `база_чеков/` и перезапись `receipt_index.json` |

Индекс хранит набор символов каждого донора — это нужно для подбора совместимого донора при генерации (поиск по `find_donor(required_chars)`).

### 4.5 Заявки (`📋 Заявки`)

Система трекинга сгенерированных чеков.

**Статусы:**
- `новый` → `в работе` → `оплачено`

**Хранение:** JSON-файлы `заявки/YYYY-MM-DD.json`

**Функции:** просмотр списка, просмотр деталей, смена статуса.

## 5. Технические возможности

### 5.1 Движок патчинга PDF (`pdf_patcher.py`)

Работает полностью на уровне байтов:
- Парсит xref-таблицу и объекты PDF вручную
- Декомпрессирует content streams (zlib)
- Заменяет CID-байты шрифта (не текст, а именно глифы)
- Пересчитывает `/Length` объектов
- Обновляет xref-таблицу и `startxref`
- Обновляет Document `/ID` (MD5 нового контента)

### 5.2 Движок CID-замены (`cid_patch_amount.py`)

- Парсит ToUnicode CMap PDF-шрифта (`beginbfchar` / `beginbfrange`)
- Кодирует произвольный текст в CID hex-строки
- При необходимости **расширяет CMap новыми символами** (кириллические гомоглифы или identity mapping)
- Поддерживает множественные замены за один проход
- Определяет оригинальный уровень zlib-сжатия и воспроизводит его

### 5.3 VTB CID-маппинг (`vtb_cmap.py`)

- Реверс-инжиниринг кастомного шрифта ВТБ
- Словарь `_CID_DIGIT` (цифры, знаки), `_CID_CYRILLIC` (кириллица)
- Некоторые заглавные буквы (Г, З, Й, К, Л, М, Н, Р, Ф, Я и др.) требуют инъекции глифов через `add_glyphs_to_13_03.py`

### 5.4 Выравнивание текста (VTB)

`vtb_patch_from_config.py` реализует **систему правого выравнивания**:
- Читает таблицу ширин глифов `/W` из CIDFontType2
- Вычисляет ширину новой строки в единицах шрифта
- Корректирует матрицу Tm (текстовую позицию) так, чтобы строка упиралась в правый «стоп»

## 7. Доступ и безопасность

- Контроль доступа через `ALLOWED_USER_IDS` (env-переменная)
- Неавторизованные пользователи получают сообщение об отказе
- Поддержка прокси: `HTTPS_PROXY` / `HTTP_PROXY` (включая proxy-auth)
- SSL-верификация отключена (кастомный urllib opener)

## 10. Итог

Бот представляет собой **полноценный инструментарий** для работы с PDF-чеками на бинарном уровне. Ключевые технические достижения:

1. **Нулевые внешние зависимости** в `bot_standalone.py` — работает на чистом Python stdlib
2. **Бинарный патчинг** без PDF-библиотек с полным сохранением структуры файла
3. **Правое выравнивание текста** через расчёт ширин CID-глифов
4. **Расширение CMap** для поддержки отсутствующих символов без пересборки шрифта
5. **Индексная база доноров** с поиском по набору символов
6. **Инъекция глифов** в VTB-шрифт для символов без собственных глифов в bfrange
